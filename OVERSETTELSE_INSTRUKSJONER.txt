OPPSUMMERING
------------
Prosjektet har fått en NO|EN-språkbryter (med flagg) i den felles
verktøylinjen, og et lite i18n-rammeverk. Standardspråk er ENGELSK; valget
huskes i localStorage (nøkkel "app_lang"). MSAT er nå FULLT oversatt. De
øvrige ekte verktøyene har foreløpig bare de felles knappene oversatt.


ARKITEKTUR (hvordan det henger sammen)
--------------------------------------
- js/i18n.js .......... Rammeverket. Gir window.I18n med:
                          t(key)      -> henter streng for gjeldende språk
                                         (fallback: engelsk -> selve nøkkelen)
                          getLang()   -> "en" | "no"
                          setLang(l)  -> bytter språk, lagrer, kjører apply()
                          apply(root) -> oversetter DOM merket med data-i18n*
                          register(lang, dict) -> språkfilene fyller ordbøker
- js/navbar.js ........ Bygger verktøylinja + NO|EN-knappen (inline SVG-flagg).
- css/navbar.css ...... Stil for språkknapp (.lang-*) og TEST-merke (.test-badge).
- js/lang/en.js, no.js  FELLES strenger (navbar + knapper som deles av alle).
- <VERKTØY>/lang/en.js, no.js  PER-VERKTØY strenger (samme mønster som MSAT).

Innlastingsrekkefølge på HVER side (viktig!):
    i18n.js  ->  js/lang/en.js, js/lang/no.js  ->  <verktøy>/lang/en.js, no.js
    ->  navbar.js
(felles_knapper.js og kalkulator-JS kan lastes før i18n.js; de bruker bare
 I18n inne i funksjoner som kjøres senere / ved klikk.)

Merking i HTML:
    data-i18n="nøkkel"              -> setter textContent
    data-i18n-html="nøkkel"        -> setter innerHTML (når strengen har <tagger>)
    data-i18n-placeholder="nøkkel" -> setter placeholder
    data-i18n-title="nøkkel"       -> setter title
I JS: window.I18n.t("nøkkel"). Bygg gjerne en lokal hjelper:
    const t = (k) => (window.I18n ? window.I18n.t(k) : k);

SLIK OVERSETTER DU ET NYTT VERKTØY (kopier MSAT-mønsteret)
----------------------------------------------------------

       I18n.register("no", { <verktøy>: { ... } });

   navbar.js (se rekkefølgen øverst). Eksempel i <VERKTØY>/*.html:
       <script src="../js/i18n.js"></script>
       <script src="../js/lang/en.js"></script>
       <script src="../js/lang/no.js"></script>

       <script src="../js/navbar.js" data-root="../"></script>
3. Statisk HTML: merk synlig tekst med data-i18n / data-i18n-placeholder /
   data-i18n-title. IKKE merk score-nedtrekk eller annet som er DATA.
4. JS-genererte strenger: bruk I18n.t("...") der teksten lages.
   - Strenger som bygges ved klikk (popup) leser automatisk gjeldende språk.
   - Strenger som bygges ved last/utregning må bygges om ved språkbytte:
       window.addEventListener('languageChanged', () => { ...bygg om... });
     (I MSAT kalles fetchConfigForProfile() på nytt – hele skjemaet bygges
      trygt om, og svar beholdes via localStorage.)

   parallell <navn>.no.json med samme ids, og la JS vise oversatt tekst med
